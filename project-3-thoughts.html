1 Guix

I have wanted to get into functional package management since last year, and I think I first tried with NixOS in January. I was immediately let down. The documentation was severly lacking. I tried to go through nix pills and quickly found a change in the example package they were using (graphviz) had completely broken the tutorial (See the issue on GitHub).

Generally, I'm not sure if functional package management is a good idea. This whole idea of patching ELF files to point to the correct dynamically linked libraries and use a dense weave of symlinked files and directories is highly offputting. At the same time, hundreds of people apparently use these systems and package managers every day as daily drivers, and their testimony indicates they're very pleased with it.

This last Summer I started getting into some of the older computers I used when I was younger. I had a lot of fun looking at the files and games I had played in my childhood, and I felt especially nostalgic for Otto Matic on the iMac G4. I've always liked the idea of modifying games, so I got the idea to try reverse engineering Otto Matic, so that I could play with it again. At the time I was lacking some important information about linking, assembly, etc. but I at least got to the point of playing with the Ghidra decompiler and even mostly reverse engineering the special file format the game used for 3D models. Although that really wasn't so much reverse engineering, as it was finding and a book and some accompanying code by Brian Greenstone about his file format. It was still pretty fun though.

Anyway I now know understand more about the fundamentals that might help with reverse engineering, so I'll try this again. But I originally set out to make 12 useful things, and reverse engineering one game won't really be that useful to anyone. So instead I'll try to learn a bit more about reversing in general in another way.

I was frustrated by the interface to Ghidra I was using over the summer. Of course, I didn't really understand the program itself, but my real issue was with just doing code-related things outside of Emacs. So I'll make a tool for reverse engineering in Emacs. That's not a very specific idea though.

Around the time I was looking at Ghidra I also saw Radare2. It looks like an excellent program but I ended up neglecting it due to what I had already invested into Ghidra. Radare2, though, is a much simpler and unix-y tool. And as a relatively simple tool, it makes sense to extend it just by creating an interface to it. So that is my more concrete idea. I will make an Emacs mode for interacting with Radare2.

Magit is an incredible Emacs interface to Git, and the library that magit uses for command interaction (transient.el) is available for use. So that's what I'll do. I'll make a magit-inspired interface to Radare2 for Emacs.
